# เอกสารสรุปฟีเจอร์และภาพรวมระบบ SAMS

**ชื่อโปรเจกต์:** SAMS (Society Application Management System) - *ระบบจัดการวัสดุสำนักงาน สหกรณ์อิสลามอัศศิดดีก*

เอกสารฉบับนี้อธิบายถึงความสามารถ โครงสร้าง และสิทธิ์การใช้งานต่างๆ ภายในระบบ SAMS ซึ่งเป็นระบบจัดการวัสดุอุปกรณ์สำนักงานที่ถูกออกแบบมาเพื่อรองรับกระบวนการทำงานตั้งแต่การตั้งคำขอเบิกจากพนักงาน ไปจนถึงกระบวนการอนุมัติและการตัดสต๊อกโดยผู้ดูแลระบบ

---

## 1. ระบบรักษาความปลอดภัยและการเข้าถึง (Authentication)
ระบบถูกออกแบบโดยใช้สถาปัตยกรรมการยืนยันตัวตนแบบ **Role-Based Access Control (RBAC)** ซึ่งแยกระดับผู้ใช้ออกเป็น 2 ระดับอย่างชัดเจน
*   **การเข้าสู่ระบบ (Login):** บังคับให้ผู้ใช้งานทุกคนเข้าสู่ระบบก่อนเพื่อระบุตัวตนและตรวจสอบสิทธิ์
*   **การแบ่งสิทธิ์ (Authorization):** แยกการแสดงผลและเมนูการใช้งาน (Sidebar & Topbar) ระหว่างผู้ใช้งานทั่วไป (User) และผู้ดูแลระบบ (Admin) ออกจากกันอย่างเด็ดขาดด้วย JSON Web Token (JWT)

---

## 2. ส่วนงานสำหรับพนักงานทั่วไป (User Features)
ระบบสำหรับพนักงานถูกออกแบบให้ใช้งานง่ายและจำลองประสบการณ์แบบการซื้อสินค้าออนไลน์ด้วย **"ระบบตะกร้า (Cart System)"** ซึ่งประกอบไปด้วย:

1.  **แคตตาล็อกวัสดุและอุปกรณ์ (Material & Equipment Browsing):**
    *   เข้าดูรายการวัสดุสิ้นเปลือง หรือ อุปกรณ์สำนักงานที่มีในคลัง
    *   ดูรายละเอียดของวัสดุ (ชื่อ, หมวดหมู่, จำนวนคงเหลือ, รูปภาพประกอบ)
2.  **ระบบตะกร้าคำขอเบิก (Cart Management):**
    *   กดหยิบวัสดุที่ต้องการลงตะกร้าจำลอง
    *   เข้าไปปรับแก้จำนวนที่ต้องการเบิก หรือ ลบรายการออกในหน้าตระกร้า
3.  **การยืนยันคำขอเบิก (Review & Submit Request):**
    *   สรุปความถูกต้องของรายการเบิกทั้งหมดก่อนที่จะกด "ส่งคำขอ (Submit)"
    *   เมื่อยืนยันแล้ว ระบบจะส่งรายการไปให้สิทธิ์การตรวจสอบของ Admin แบบอัตโนมัติ
4.  **ระบบติดตามสถานะ (Request Tracking):**
    *   เข้าดูสถานะปัจจุบันของคำขอเบิก (เช่น รอดำเนินการ, อนุมัติแล้ว, ถูกปฏิเสธ)
    *   เรียกดูรายละเอียดเชิงลึกของคำขอนั้นๆ ได้ตลอดเวลา
5.  **ประวัติส่วนตัว (History & Profile):**
    *   ติดตามประวัติการทำรายการเบิกจ่ายของตนเองที่ผ่านมา
    *   เข้าดูข้อมูลและจัดการโปรไฟล์ส่วนบุคคล

---

## 3. ส่วนงานสำหรับผู้ดูแลระบบ (Admin Features)
สิทธิ์ของ Admin จะได้รับเมนูจัดการเฉพาะ เพื่อใช้สำหรับควบคุมสินค้าคงคลังและพิจารณาคำขอเบิก ซึ่งเป็นส่วนสำคัญ (Core) ของระบบ:

1.  **แดชบอร์ดสรุปผล (Admin Dashboard):**
    *   แสดงข้อมูลภาพรวม สถิติคำขอ และยอดสรุปต่างๆ ภายในคลัง
2.  **ระบบจัดการวัสดุ (Material Management - CRUD):**
    *   ดูรายชื่อวัสดุทั้งหมดในระบบ
    *   เพิ่มวัสดุอุปกรณ์รายการใหม่เข้าสู่คลัง
    *   แก้ไข เปลี่ยนแปลงรายละเอียดสินค้า (หมวดหมู่, ชื่อ, หน่วยนับ, ขนาดบรรจุ)
3.  **ระบบบริหารสต๊อก (Inventory & Stock Control):**
    *   แสดงปริมาณและควบคุมจำนวนคงเหลือของสต๊อกในปัจจุบัน (Inventory)
    *   ดูประวัติสรุปการเข้า-ออกของการเบิกจ่ายและเติมสต๊อก (Inout History)
4.  **ระบบอนุมัติคำขอเบิก (Request Approval Workflow):**
    *   รับรายการแจ้งเตือนคิวเบิกวัสดุจากพนักงาน (Request List)
    *   พิจารณารายละเอียดการเบิก และสามารถกดเพื่อดำเนินการพิจารณา **อนุมัติ (Approve)** หรือ **ปฏิเสธ (Reject)** คำขอ
5.  **ระบบการจัดการพนักงาน (User Management):**
    *   ดูรายชื่อและข้อมูลของพนักงานทั้งหมด
    *   เพิ่ม ระงับสิทธิ์ หรือแก้ไขข้อมูลของพนักงานภายในคลังระบบ

## 4. โครงสร้างและสถาปัตยกรรมทางเทคนิค (Tech Stack)

### 🖥️ Frontend (ระบบหน้าบ้าน)
พัฒนารูปแบบ Single Page Application เพื่อตอบสนองที่รวดเร็ว
*   **Framework:** React.js
*   **Routing:** React Router v6 สำหรับแบ่งหน้าหลักของระบบ เช่น `/login`, `/admin/dashboard`, `/borrow-material`
*   **State Management:** React Context API (สำหรับจัดการสถานะของในตะกร้าระหว่างทำรายการ)
*   **Styling:** Tailwind CSS สำหรับออกแบบ UI ให้ทันสมัย มีความเป็นระเบียบและ Responsive

### ⚙️ Backend (ระบบหลังบ้าน)
พัฒนาระบบฝั่งความปลอดภัยและจัดการฐานข้อมูล
*   **Framework:** Python ใช้งานร่วมกับ FastAPI ทำให้ระบบ API รองรับการทำงานแบบ Asynchronous ได้อย่างมีประสิทธิภาพ 
*   **Data Validation:** Pydantic (`backend/schemas`) ช่วยตรวจสอบและยืนยันความถูกต้องของข้อมูล (Schema) ที่ได้รับ
*   **Database ORM:** SQLAlchemy (`backend/models`) จัดการโครงสร้างฐานข้อมูล คิวรี่ดึงข้อมูล และบันทึกประวัติเข้าระบบ
*   **Security:** ป้องกัน Endpoint เบื้องหลังด้วย CORS และเข้ารหัสโทเคนผ่านฟังก์ชันของ `auth.py`
